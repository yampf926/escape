# QTERW REBOOT DESIGN

## 1. 프로젝트 리부트 개요

### 프로젝트명
- QTERW: MEMORY CENSOR

### 한 줄 콘셉트
- 삭제된 기억을 복원해 탈출하되, 복원할수록 존재가 붕괴되는 2D 심리공포 잠입 게임.

### 장르
- 2D Top-Down Psychological Horror
- Stealth + Investigation + Choice-Driven Narrative

### 플랫폼/기술
- Java Swing 프로토타입 -> 추후 엔진 이관(권장: Godot/Unity)
- 목표 프레임: 60 FPS
- 해상도: 1:1 기본(1100x1100), 내부 월드는 스크롤 방식

---

## 2. 코어 게임 루프

1. 구역 진입
2. 단서/기억 조각 수집
3. 감시 시스템 우회
4. 복원/은폐 선택 수행
5. 상태값 변화(노출도/안정도/기억밀도)
6. 탈출 시도 또는 강제 종료

플레이어는 매 챕터에서 최소 1회 이상 "대가 있는 선택"을 해야 한다.

---

## 3. 핵심 시스템 명세

### 3.1 플레이어 동작
- 이동: WASD
- 상호작용: 마우스 클릭
- 기록 확인: J
- 일시정지 메뉴: ESC

### 3.2 자원 3종
- 노출도(Exposure): 감시/경보 축적 수치
- 안정도(Stability): 정신적 붕괴 수치
- 기억밀도(Memory Density): 진실 접근 수치

### 3.3 공통 승패 규칙
- 승리 기본: 챕터 목표 달성 + 출구 도달
- 패배 공통:
  - 노출도 100% -> 체포/삭제
  - 안정도 0% -> 개체 붕괴
  - 시간 제한 초과 -> 강제 검열 종료

### 3.4 감시/잠입 규칙
- 카메라 시야 콘에 노출 시 노출도 증가
- 감시 드론/경비 개체는 소리 이벤트 반응
- 전력 차단 시 카메라 무력화, 대신 비상등 점등(환영 출현율 증가)

### 3.5 기억 복원 규칙
- 기억 조각은 3등급:
  - 파편(Fragment): 안정도 소모 적음, 정보량 적음
  - 핵심(Core): 안정도 소모 중간, 정보량 큼
  - 금지(Forbidden): 안정도 급감, 진실 분기 개방

### 3.6 공포 연출 규칙
- 점프스케어 제한: 챕터당 최대 1회
- 주요 연출은 환경 변화 기반:
  - 동일 방의 구조 재배열
  - 텍스트 오염(안내문 변질)
  - 오디오 방향성 혼선
  - NPC 실루엣 중복 출현

### 3.7 선택 시스템
- 복원(Truth): 기억밀도↑, 안정도↓, 위험 분기 개방
- 은폐(Survival): 노출도↓, 기억밀도↓, 생존 분기 강화
- 고정 정답 없음: 누적 선택 비율로 엔딩 타입 계산

## 4. 챕터 구성(4+1)

## CHAPTER 1 - 검사동 프롤로그
- 목적: 시스템 튜토리얼 + 1차 선택
- 맵: 검사실, 관찰실, 복도
- 목표:
  - 검열 로그 2개 확보
  - 의식 안정 검사 결과를 조작(복원/은폐 선택)
- 위협: 고정 카메라 + 저속 순찰 개체
- 보스 규칙 없음

## CHAPTER 2 - 기록 보관소
- 목적: 이름이 지워진 대상의 실체 추적
- 맵: 문서고, 파쇄실, 승인실
- 목표:
  - 문서 키 3개 수집
  - 파쇄 대기열에서 기록 1건 구조 또는 삭제
- 위협: 소음 반응 경비 시스템
- 특수: 조명 차단 시 지형 일부 변경

## CHAPTER 3 - 병동 구역
- 목적: 살아있는 사망자 명단 확인
- 맵: 병실 블록, 간호 스테이션, 냉동 보존실
- 목표:
  - 생체 로그/사망 로그 불일치 원인 규명
  - 환자 ID 한 명을 해방 또는 봉인
- 위협: 환영 개체(시야 내/외 판정이 다름)
- 특수: 안정도 40% 이하에서 오브젝트 라벨 교란

## CHAPTER 4 - 근무 통제실
- 목적: 시설 운영 알고리즘 재작성
- 맵: 서버실, 모니터링 홀, 권한 코어
- 목표:
  - 전력 라우트 재배선
  - 감시 알고리즘 모드 선택(은폐형/강제형)
- 위협: 고속 추적자 + 타이머 압박
- 특수: 잘못된 라우팅 시 즉시 경보 레벨 상승

## CHAPTER 5 - 최종 국면(경계층)
- 목적: 플레이어 정체 확정
- 맵: 기억 재구성 공간(동적 맵)
- 목표:
  - 누적 선택 결과 기반 마지막 결단
- 위협: 시스템 자체(규칙이 실시간으로 변형)
- 특수: UI 거짓 정보 섞임

## 5. 엔딩 설계

### E1 은폐 생존
- 조건: 노출도 평균 낮음, 기억밀도 낮음, 안정도 높음
- 결과: 살아남지만 진실 소실

### E2 진실 도달
- 조건: 기억밀도 높음, 안정도 중간 이상
- 결과: 시설의 실체 공개, 고통스러운 생존

### E3 시스템 동화
- 조건: 효율 위주의 통제 선택 다수
- 결과: 플레이어가 새 검열 관리자 역할로 통합

### E4 개체 오염
- 조건: 안정도 붕괴 + 금지 기억 과다 복원
- 결과: 인간 인식 불가 상태로 재분류

## 8. 개발 마일스톤

## M1 프로토타입(1주)
- 이동/상호작용/카메라/충돌
- 노출도/안정도/기억밀도 HUD
- 챕터 1 플레이 가능

## M2 수직 슬라이스(2~3주)
- 챕터 2~3 추가
- 추적자/카메라/환영 시스템
- 저장/불러오기

## M3 알파(4~6주)
- 챕터 4~5 구현
- 엔딩 4종 연결
- 밸런싱/버그 수정

## M4 베타
- 아트/사운드 리파인
- 퍼포먼스 최적화
- 사용자 테스트 반영

---

## 9. 즉시 구현 우선순위

1. ChapterDef JSON 로더 도입
2. 시스템 3종(노출/안정/기억) 독립 모듈화
3. 챕터 1 재구현(튜토리얼 + 첫 선택)
4. 경비 시야 + 그림자 추적자 분리
5. 엔딩 계산기(선택 로그 기반)
